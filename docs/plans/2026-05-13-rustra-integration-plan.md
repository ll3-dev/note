# Rustra Integration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the HTTP bridge (note-engine Axum server) with rustra `#[command]` + `@rustra/bun` FFI adapter.

**Architecture:** `webview → RPC → Bun → rustra EngineClient → FFI (C ABI) → note-bridge (rustra dispatch) → note-core → SQLite`. The `note-engine` crate is deleted and replaced by `note-bridge` which exposes a `rustra_note_invoke` C function. Bun calls this via FFI. TypeScript types are generated by rustra CLI.

**Tech Stack:** Rust (rustra, schemars, rusqlite, serde), TypeScript (@rustra/bun, @rustra/cli), Electrobun

The `MoveBlockInput` and `MoveBlocksInput` from note-core use the internal `ParentBlockIdInput` enum which can't be deserialized simply. The bridge needs its own input types that use `Option<String>` for `parentBlockId` and translate to `ParentBlockIdInput::Set(...)`.

The bridge holds a `Connection` in a `Mutex` (std::sync, not tokio) and passes it through to note-core functions.

```rust
use std::path::PathBuf;
use std::sync::Mutex;

use rustra::prelude::*;
use rusqlite::Connection;
use serde_json::Value;

// -- Input/Output types for commands that need different shapes --

#[derive(Deserialize, JsonSchema)]
#[serde(rename_all = "camelCase")]
struct GetPageDocumentInput { page_id: String }

#[derive(Deserialize, JsonSchema)]
#[serde(rename_all = "camelCase")]
struct SearchPagesInput { query: String, limit: Option<usize> }

#[derive(Deserialize, JsonSchema)]
#[serde(rename_all = "camelCase")]
struct SearchWorkspaceInput { query: String, limit: Option<usize> }

#[derive(Deserialize, JsonSchema)]
#[serde(rename_all = "camelCase")]
struct ListBacklinksInput { page_id: String }

#[derive(Serialize, JsonSchema)]
struct DeleteBlockOutput { deleted: bool }

#[derive(Serialize, JsonSchema)]
#[serde(rename_all = "camelCase")]
struct EngineInfoOutput {
    engine_version: String,
    api_version: String,
    capabilities: Vec<&'static str>,
}

// -- State --

pub struct BridgeState {
    connection: Mutex<Connection>,
    database_path: PathBuf,
}

impl BridgeState {
    pub fn new(user_data_path: PathBuf) -> rustra::Result<Self> {
        let connection = note_core::database::open_database(&user_data_path)
            .map_err(|e| RustraError::internal(e.to_string()))?;
        Ok(Self {
            database_path: user_data_path.join("note.sqlite3"),
            connection: Mutex::new(connection),
        })
    }
}

// -- Command functions --

#[command]
fn engine_info(_input: EmptyInput) -> Result<EngineInfoOutput> { ... }

#[command]
fn database_status(state: &BridgeState, _input: EmptyInput) -> Result<note_core::database::DatabaseStatus> { ... }

// ... (all 23 commands)

pub fn note_package() -> Package {
    register!(Package::builder("com.note"),
        engine_info,
        database_status,
        list_pages,
        list_archived_pages,
        get_page_document,
        create_page,
        update_page,
        move_page,
        delete_page,
        restore_page,
        purge_expired_archived_pages,
        undo_page_history,
        redo_page_history,
        create_block,
        create_blocks,
        update_block,
        delete_block,
        delete_blocks,
        move_block,
        move_blocks,
        search_pages,
        search_workspace,
        list_backlinks,
    ).build()
}
```

**Important**: The `#[command]` macro in rustra requires `fn(input: I) -> Result<O>`. But note-core functions need `&Connection`. The bridge commands must capture `BridgeState` via a closure pattern. Since `Package::command()` accepts `Fn(I) -> Result<O>`, we register each command as a closure that locks the mutex:

```rust
let state = BridgeState::new(user_data_path)?;
let package = Package::builder("com.note")
    .command("listPages", move |input: EmptyInput| {
        let conn = state.connection.lock().map_err(|e| RustraError::internal(e.to_string()))?;
        note_core::documents::list_pages(&conn)
            .map_err(|e| map_database_error(e))
    })
    // ... more commands
    .build();
```

For commands needing `&mut Connection`, use `state.connection.lock().unwrap()` and pass `&mut *conn`.

Error mapping function:

```rust
fn map_database_error(error: rusqlite::Error) -> RustraError {
    match error {
        rusqlite::Error::QueryReturnedNoRows => RustraError::custom("notFound", "requested record was not found"),
        rusqlite::Error::InvalidQuery => RustraError::custom("invalidRequest", "request violates note engine domain rules"),
        _ => RustraError::custom("databaseError", error.to_string()),
    }
}
```

**Step 1: Add FFI entry point to lib.rs**

Following the calculator example pattern, add `rustra_note_invoke` and `rustra_note_free_string` as `#[unsafe(no_mangle)] extern "C"` functions.

The invoke function:
1. Parses incoming JSON `{ "command": string, "args": object, "userDataPath": string }`
2. On first call, initializes `BridgeState` with the `userDataPath`
3. Calls `note_package().invoke_json(command, args)`
4. Returns `{ "ok": true, "result": ... }` or `{ "ok": false, "error": ... }`

For state management, use `std::sync::OnceLock<BridgeState>` or pass userDataPath in every call. Since the state (Connection) must be shared across calls, use a `static` with `OnceLock`:

```rust
use std::sync::OnceLock;

static STATE: OnceLock<BridgeState> = OnceLock::new();
```

The first invoke with `userDataPath` initializes it.

### Task 7: Rewrite engineProcess.ts as FFI bridge

**Files:**
- Modify: `src/bun/engine/engineProcess.ts`

**Step 1: Rewrite to use FFI**

Replace HTTP process spawn with Bun FFI loading of the note-bridge library:

```ts
import { dlopen, FFIType, suffix } from "bun:ffi";
import { existsSync } from "node:fs";
import path from "node:path";

export type EngineProcess = {
  invoke: (command: string, args: unknown) => unknown;
  stop: () => void;
};

export function startEngineProcess(
  userDataPath: string
): EngineProcess {
  const libraryPath = resolveEngineLibraryPath();

  const library = dlopen(libraryPath, {
    rustra_note_invoke: {
      args: [FFIType.cstring],
      returns: FFIType.cstring,
    },
    rustra_note_free_string: {
      args: [FFIType.cstring],
      returns: FFIType.void,
    },
  });

  return {
    invoke(command: string, args: unknown) {
      const payload = JSON.stringify({ command, args, userDataPath });
      const resultPtr = library.symbols.rustra_note_invoke(payload);
      const resultJson = resultPtr;
      library.symbols.rustra_note_free_string(resultPtr);

      const parsed = JSON.parse(resultJson);
      if (!parsed.ok) {
        throw Object.assign(new Error(parsed.error), parsed);
      }
      return parsed.result;
    },
    stop() {
      // No-op: library unloads on process exit
    },
  };
}

function resolveEngineLibraryPath(
  env: NodeJS.ProcessEnv = process.env,
  baseDir = process.cwd()
) {
  const ext = suffix;
  const candidates = [
    env["NOTE_BRIDGE_LIBRARY"],
    path.resolve(baseDir, `target/debug/libnote_bridge${ext}`),
    path.resolve(baseDir, `bin/libnote_bridge${ext}`),
  ].filter(Boolean);

  const found = candidates.find((p) => existsSync(p));
  if (!found) {
    throw new Error(`note-bridge library not found. Checked: ${candidates.join(", ")}`);
  }
  return found;
}
```
